## Local setup

### Prerequisites

- [Node.js](https://nodejs.org/) (LTS)

### 1. Install dependencies

```bash
cd taskflow
npm install
```

> **Windows PowerShell:** If `npm` is blocked by execution policy, use `npm.cmd` instead (e.g. `npm.cmd install`).

### 2. Start the REST API (port 3000)

```bash
npm run api
```

API base: `http://localhost:3000/tasks`

### 3. Serve the frontend (do not open `index.html` as a file)

In a **second** terminal:

```bash
npm run serve
```

> **Important:** Do not double-click `index.html`. CORS requires a local server so `fetch()` can reach JSON Server.

Then open **http://localhost:5500** in your browser.

> Use a local server so `fetch()` to JSON Server works (CORS / security).

### 4. Exam ŌĆö point frontend to production API

In `js/app.js`, change the `API` constant from localhost to your deployed API URL, for example:

```javascript
const API = 'https://your-api.onrender.com/tasks';
```

Redeploy the frontend after this change.

## REST API contract

| Method | Endpoint       | Purpose        |
|--------|----------------|----------------|
| GET    | `/tasks`       | Read all tasks |
| POST   | `/tasks`       | Create task    |
| PUT    | `/tasks/:id`   | Update task    |
| DELETE | `/tasks/:id`   | Delete task    |

## Task data model

| Field | Type | Values |
|-------|------|--------|
| id | number | Generated by server |
| title | string | Required |
| description | string | Optional |
| priority | string | `low`, `medium`, `high` |
| status | string | `pending`, `in-progress`, `done` |
| dueDate | string | ISO date (`YYYY-MM-DD`) |
